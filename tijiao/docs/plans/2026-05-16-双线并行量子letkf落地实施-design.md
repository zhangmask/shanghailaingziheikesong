# 双线并行量子 LETKF 落地实施 Design

## 目标

将 `lunwenfenxi/2` 中值得吸收的方法精华，转化为当前项目可执行、可对比、可止损的双线并行实验设计。

本设计不追求一次性重做整条主线，而是拆成：

- 一条短期稳妥线，优先争取比当前量子版本更接近经典 `fixed`
- 一条中期创新线，优先验证量子核是否更适合进入表示层/降阶层

## 背景

- `实验 4` 已证明：量子核直接写入观测对角加权会显著退化。
- `实验 5` 已证明：量子核弱混合更稳定，但仍未超过经典 `fixed`。
- `实验 6` 已尝试把量子特征改为“可信度特征”，说明特征层可以继续优化。
- `实验 7` 已把量子核前移到表示层分组，说明“接入位置”可能比“核本身有没有结构”更关键。

这意味着当前问题已经从“量子核能否算出来”转为：

> 量子信息应该以什么形式、在同化流程的哪一层进入，才可能既稳定又有收益。

## 核心判断

`lunwenfenxi/2` 最值得吸收的，不是整套理论原样实现，而是三个工程上可落地的思想：

- 量子核用于表达局部结构，而不必直接替代最终权重
- 降阶 / 分组 / 子块组织，比单点点权重更接近“非交换条件期望”的工程近似
- 稀疏量子结构信号应交给经典稳健更新去完成最终分析，而不是让量子部分直接碰最脆弱的逆矩阵步骤

因此，推荐采用双线并行：

- `短期稳妥线`：量子核只做局部 `block` 级结构调制
- `中期创新线`：量子核先做局部观测分组 / 降阶，再由经典规则更新

## 总体架构

采用 `共享骨架 + 双线实验 + 统一比较层` 的结构。

### 共享骨架

复用当前 `LETKF` 稳定主流程：

- 数据读取与预测输出
- `Lorenz96 + RK4`
- `sym_sqrt`
- `stable_inverse`
- `train/test_1` 指标统计
- `summary.json/csv`、`best_test_prediction.csv`、`result.csv`

### 双线实验

- `exp8_quantum_block_mix_letkf.py`
  - 基于 `exp5`
  - 负责短期稳妥线
- `exp9_quantum_reduced_group_letkf.py`
  - 基于 `exp7`
  - 负责中期创新线

### 统一比较层

- `compare_quantum_variants.py`
  - 只负责比较不同实验产物
  - 不承载算法细节

## 设计原则

- 不在第一轮就做“大一统超级脚本”
- 尽量保留现有 `exp5 / exp6 / exp7` 的实验谱系
- 量子模块优先负责“结构组织”，经典模块优先负责“数值稳健更新”
- 一次只增加一类新变量，保证失败时能定位原因
- 所有新方法都必须与 `fixed`、`corr`、`exp5` 最优版本直接对照

## 短期稳妥线

### 脚本

`exp8_quantum_block_mix_letkf.py`

### 目标

在不明显增加数值风险的前提下，比 `实验 5` 的最佳量子混合更接近经典 `fixed`，并争取在局部优于旧量子混合版本。

### 方法主张

量子模块不再直接输出单观测点最终权重，而是先输出局部 `block` 级结构信号，例如：

- 局部观测块之间的相似度
- 观测块的可信度摘要
- block 内部一致性或稳定性分数

再将该结构信号作为经典先验权重的弱调制项：

- `distance_prior * weak_quantum_block_signal`
- 或 `corr_prior * weak_quantum_block_signal`

### 接入位置

位于经典局部权重构造与最终 `R^{-1}` 之间，但只做弱混合，不直接决定最终更新强度。

### 第一版范围

- 只做一种 `block` 结构信号
- 只扫小 `lambda`
- 只保留最小必要对照

### 预期收益

- 保留 `exp5` 的稳定骨架
- 避免量子信号直接冲击单点权重
- 更容易回答“量子结构调制是否比量子单点权重更合理”

## 中期创新线

### 脚本

`exp9_quantum_reduced_group_letkf.py`

### 目标

把 `lunwenfenxi/2` 中“降阶 / 分组 / 子块组织”的思想翻译成工程可跑版本，验证量子核是否更适合在表示层发挥作用。

### 方法主张

量子模块不直接决定观测权重，而是先回答：

- 哪些观测应该视作同一类局部证据
- 哪些观测可以先归并后再更新
- 哪些局部结构更值得保留

然后：

1. 基于 `obs-obs` 量子相似度做分组
2. 每组内部仍采用经典稳健更新
3. 再做跨组合并，得到状态点最终分析结果

### 接入位置

位于 `local_obs_idx` 与局部分析更新之间，先改变局部证据组织方式，再做经典分析。

### 第一版范围

- 只做 `2-3` 组的最小分组版
- 阈值分组或贪心分组二选一
- 组内用 `fixed`
- 跨组合并规则只用最简单版本

### 预期收益

- 更贴近“结构先行、更新后置”的思路
- 比直接量子权重更稳
- 即使误差不立刻超过 `fixed`，也能输出更有解释力的结构证据

## 共享模块划分

不建议一开始就大拆文件，但应按职责组织函数边界。

### 骨架模块

- 数据读取
- Lorenz96 推进
- 稳定逆与平方根
- 评测与结果导出

### 量子特征模块

统一形成局部量子特征接口，支持：

- 窗口数值特征
- 可信度特征
- 观测相似度特征

### 结构组织模块

负责：

- block 摘要
- 分组
- 子块筛选
- 组内摘要
- 跨组合并

### 接入策略模块

负责定义量子信息如何进入：

- 权重弱混合
- 表示层分组
- 子块降阶

## 数据流

### exp8 数据流

1. 输入某时间步 `ensemble` 与 `obs_vec`
2. 构造经典局部上下文
3. 构造量子 `block` 结构信号
4. 将量子信号压回观测点层作为弱调制项
5. 仍走经典局部分析公式

本质上是：

> 量子信号做结构调制，经典流程做数值更新

### exp9 数据流

1. 输入局部观测集合 `local_obs_idx`
2. 构造 `obs-obs` 量子相似度矩阵
3. 做局部分组或降阶
4. 每组内执行经典稳健更新
5. 跨组合并为最终分析结果

本质上是：

> 量子模块决定证据组织，经典模块决定更新数值

## 统一比较层

`compare_quantum_variants.py` 至少比较：

- `fixed`
- `corr`
- `exp5` 最佳
- `exp8` 最佳
- `exp9` 最佳

输出：

- 总表
- 最佳方案排行
- 相对 `fixed` 的 RMSE / MAE 差值
- 稳定性对比结论

## 实验顺序

### 阶段 A：先做 exp8

- 先验证 `block` 级弱调制是否比 `exp5` 更稳
- 仅扫小 `lambda`
- 不同时引入多个 block 定义

### 阶段 B：再做 exp9

- 先做最小分组版
- 先固定 `2-3` 组与简单合并规则
- 先回答“是否比权重层量子接入更合理”

### 阶段 C：最后才允许交叉吸收

只有当 `exp8` 或 `exp9` 任一方向出现积极信号时，才考虑做：

- 可信度特征 + 分组
- block 调制 + 分组
- 更复杂的降阶结构

## 通过标准

### exp8 通过标准

- 不出现明显数值爆炸
- 比 `exp5` 最佳更稳或更接近 `fixed`
- 至少在局部时间段表现出可解释改进

### exp9 通过标准

- 比直接量子权重接入更稳
- 能给出有解释力的分组统计
- 即使未超过 `fixed`，也能说明表示层前移比权重层直接接入更合理

## 止损条件

### exp8 止损

- 全部 `lambda` 都比 `exp5` 最佳更差
- 波动明显更大
- block 摘要本质上退化成噪声

### exp9 止损

- 分组近似随机
- 分组持续显著劣化误差
- 分组结构没有任何解释力
- 复杂度上升但没有任何局部收益

## 风险控制

- 所有新接入都保留 `stable_inverse`
- 保持 `lambda` 小步扫描
- 先固定一类特征，不同时更换接入层和特征层
- 所有结果都回写 `jindu.md`
- 不碰其他人正在开发的目录，只在本次实验脚本与计划文档范围内推进

## 预期交付

- `exp8_quantum_block_mix_letkf.py`
- `exp9_quantum_reduced_group_letkf.py`
- `compare_quantum_variants.py`
- 对应实验文档
- `jindu.md` 关键记录
- 一份可直接对外说明“为什么这样设计”的方法叙事

## 一句话结论

本设计的核心不是“继续把量子核往权重里塞”，而是：

> 把量子核从脆弱的数值更新层，前移到更适合它发挥作用的结构组织层，同时保留经典 `LETKF` 的稳健更新骨架。
