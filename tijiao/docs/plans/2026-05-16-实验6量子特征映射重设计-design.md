# 实验6量子特征映射重设计 Design

## 目标

在不改变 `LETKF` 主骨架的前提下，重新设计量子特征映射，使量子核不再主要表达“局部窗口相似度”，而是更贴近“观测对当前局部更新是否可信”的信息结构。

## 背景

- `实验 3` 已经证明当前浅层量子特征映射可以生成有结构的量子核矩阵。
- `实验 4` 证明当前量子核直接做观测对角加权会显著退化。
- `实验 5` 证明温和混合比完全替代更稳，但仍然没有超过 `fixed` 基线。

因此，当前首要怀疑对象不再只是“接入位置”，还包括“量子核输入特征本身不对”。

当前映射的问题是：

- 输入主要来自状态窗口与观测窗口的局部数值片段；
- 它更容易表达“像不像”；
- 但 `LETKF` 需要的更像是“这个局部观测现在值不值得信”和“它对分析更新是不是有帮助”。

## 核心假设

> 如果量子特征直接编码与观测可信度更相关的统计量，而不是只编码局部数值相似度，那么得到的量子核将更贴近同化更新所需的信息结构，并可能比当前量子核更稳定、更有用。

## 设计原则

- 保留 `LETKF` 主骨架；
- 不改 `RK4`、集合传播、`sym_sqrt`、局部更新流程；
- 先只替换量子特征定义，不同时改接入位置；
- 保持与 `实验 5` 尽量可比，避免一次改太多；
- 优先使用当前集合和观测中已经能稳定提取的统计量。

## 特征设计

### 方案 A：可信度导向特征

对每个局部状态点 `i` 与局部观测点 `j`，构造以下特征：

1. 局部创新幅值  
   `|y_j - \bar{Hx}_j| / (\sigma_{obs} + eps)`

2. 集合离散度  
   `std(Hx_j^{(n)}) / (\sigma_{obs} + eps)`

3. 状态-观测耦合强度  
   使用当前 `x_pert[:, i]` 与 `y_pert[:, j]` 的绝对相关性或协方差摘要

4. 环距离先验  
   `cyclic_distance(i, j) / support_radius`

这组特征不再主要问“局部窗口像不像”，而是问：

- 这个观测当前偏差大不大；
- 当前集合对这个观测稳不稳；
- 这个状态和观测有没有更新耦合；
- 距离先验是否支持它参与当前局部更新。

### 方案 B：窗口可信度摘要特征

在 `j` 周围取一个小窗口，对上述量做摘要，例如：

- 局部创新均值
- 局部创新方差
- 局部离散度均值
- 局部耦合强度均值

这个方案更平滑，但信息也更粗。

## 推荐实现

优先实现 **方案 A**。

原因：

- 更接近单观测是否可信的粒度；
- 最容易与当前 `LETKF` 局部更新直接对应；
- 更适合和 `实验 5` 的权重逻辑做公平对照。

## 接入方式

为了保证变量单一，`实验 6` 不改变量子核所在层级，只做最小验证：

1. 构造新的量子核 `K_q^{cred}`
2. 先接入现有最稳的弱混合框架：

```math
w = (1 - \lambda) + \lambda K_q^{cred}
```

只扫描小范围：

- `lambda = 0.02, 0.05, 0.08, 0.10, 0.12, 0.15`

这样可以避免重复 `实验 5` 中大 `lambda` 的强退化。

## 对照组

至少保留：

- `fixed`
- `corr`
- `exp5 linear_quantum_mix` 最优版本
- `exp6 credibility_quantum_mix`

## 输出

### 实验层

- `train/test_1 RMSE`
- `train/test_1 MAE`
- 各 `lambda` 汇总表

### 结构层

- 新旧量子核统计量对比
- 新旧量子核热图对比
- 新旧量子权重分布对比

## 通过标准

以下任一情况即可认为 `实验 6` 有价值：

1. 新特征映射下的量子混合结果超过 `实验 5` 最优混合版本；
2. 新特征映射在更小 `lambda` 区间内更稳定；
3. 即使未超过 `fixed`，也能明显减少“量子权重一增强就失稳”的问题。

## 止损条件

若出现以下情况，则说明问题不主要在特征定义：

- 新量子核统计结构与旧版几乎无差别；
- 小 `lambda` 下仍明显不如 `实验 5` 最优混合；
- 一旦进入更新层仍快速退化。

此时应把重点转移到接入层，而不是继续打磨特征。

## 当前结论预期

`实验 6` 的目标不是立刻打赢 `fixed`，而是先回答：

> 当前失败到底是不是“量子核表达错了”。

如果答案是“有改善”，则后续再把这个新特征映射带入更前移的表示层实验。
