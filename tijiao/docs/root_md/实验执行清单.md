# 实验执行清单

> 更新时间：2026-05-16
> 目的：把“探索优先的小实验设计”进一步收缩成可直接执行的清单，确保每一步都有明确输入、输出、判断标准和止损条件。

---

## 一、使用方式

这份清单不是讨论性文档，而是执行文档。

使用原则：

1. 严格按顺序执行，不跳步；
2. 每完成一步，必须记录结果；
3. 每一步都要先回答：
   - 这个实验验证什么假设？
   - 结果支持继续还是止损？
4. 没有通过上一阶段，不进入下一阶段。

---

## 二、总执行顺序

按如下顺序执行：

1. `实验 0`：固定统一基线
2. `实验 1`：固定局部化敏感性
3. `实验 2`：经典代理核
4. `实验 3`：量子核结构
5. `实验 4`：量子核接入 LETKF

---

## 三、实验 0：固定统一基线

### 3.1 目标

建立后续所有实验共同参照的基线版本，避免后面对比口径漂移。

### 3.2 输入

- 数据：
  - `lorenz96_train.csv`
  - `lorenz96_test_1.csv`
- 当前最优经典/量子调参参数与结果

### 3.3 需要固定的版本

#### 经典强基线

- 方法：`LETKF`
- 参数：
  - `ens_size = 40`
  - `infl = 1.02`
  - `loc_radius = 20`
  - `obs_std = 0.5`
  - `dt = 0.05`
  - `forcing = 8.0`
  - `seed = 2`

#### 当前量子调参版

- 方法：`LETKF + 量子参数调优`
- 参数：
  - `ens_size = 40`
  - `infl = 1.02`
  - `loc_radius = 20`
  - `obs_std = 0.5`
  - `dt = 0.05`
  - `forcing = 8.0`
  - `seed = 0`

### 3.4 输出

需要整理成一张固定对照表：

| 方案 | train RMSE | train MAE | test_1 RMSE | test_1 MAE |
|------|------------|-----------|-------------|------------|
| 朴素基线 |  |  |  |  |
| 经典强基线 |  |  |  |  |
| 量子调参版 |  |  |  |  |

### 3.5 判断标准

只有当：

- 结果文件齐全；
- 指标口径固定；
- 后续都能复用同一基线；

才算通过。

### 3.6 止损条件

如果当前基线结果都还不稳定，就不要继续做更深实验，先回去修基础流程。

---

## 四、实验 1：固定局部化敏感性实验

### 4.1 目标

验证固定经验局部化是否真的显著影响当前误差表现。

### 4.2 要验证的假设

> 假设 1：固定局部化半径是当前误差的重要来源之一。

### 4.3 输入

- 方法：`LETKF`
- 固定参数：
  - `ens_size = 40`
  - `infl = 1.02`
  - `obs_std = 0.5`
  - `dt = 0.05`
  - `forcing = 8.0`
- 只改变：
  - `loc_radius`

### 4.4 扫描范围

建议最少扫描：

- `loc_radius = 4, 6, 8, 10, 15, 20, 25, 30`

### 4.5 输出

整理成表格：

| loc_radius | train RMSE | test_1 RMSE | train MAE | test_1 MAE | 备注 |
|------------|------------|-------------|-----------|------------|------|

并额外输出：

- 分时间步 RMSE 曲线
- 分维度 RMSE 曲线

### 4.6 通过标准

若出现以下任一现象，则支持继续：

1. `loc_radius` 对误差影响显著；
2. 不同时间段最优半径不同；
3. 固定单一半径无法兼顾所有局部情况。

### 4.7 止损条件

若发现：

- 不同半径结果差异极小；
- 固定局部化基本不敏感；

则“局部化是主要瓶颈”不成立，暂停后续量子核主线。

---

## 五、实验 2：经典代理核实验

### 5.1 目标

验证“数据驱动权重”是否值得替代“固定经验局部化”。

### 5.2 要验证的假设

> 假设 2：数据驱动相似度权重可以优于固定经验局部化。

### 5.3 输入

- 方法：`LETKF`
- 主骨架保持不变
- 将固定 `W_i` 替换为经典代理核构造的 `W_i^(cls)`

### 5.4 建议先试的 3 类代理核

#### A. 扰动相似度核

- 输入特征：
  - 局部集合扰动摘要
- 形式：
  - RBF 核 / 高斯核

#### B. 创新相关核

- 输入特征：
  - innovation
  - 局部观测残差

#### C. 混合核

- 输入特征：
  - 空间距离
  - 扰动统计
  - innovation 摘要

### 5.5 输出

至少要输出：

| 核类型 | train RMSE | test_1 RMSE | train MAE | test_1 MAE | 是否稳定 |
|--------|------------|-------------|-----------|------------|----------|

并保存：

- 若干时间步的权重矩阵热图
- 若干局部点的观测权重曲线

### 5.6 通过标准

若满足以下任一条件，则通过：

1. 经典代理核稳定优于固定局部化；
2. 经典代理核虽提升有限，但显著表现出更合理、更自适应的权重结构；
3. 在不同时间段上比固定半径更灵活。

### 5.7 止损条件

若经典代理核：

- 明显更差；
- 或数值不稳定；
- 或本质上退化回固定半径；

则当前量子核替代路线暂停，不继续往量子版本推进。

---

## 六、实验 3：量子核结构实验

### 6.1 目标

不急着接入主流程，先判断量子特征映射构造出来的核矩阵是否有用。

### 6.2 要验证的假设

> 假设 3：量子特征映射核在小规模下能生成有结构的相似度矩阵。

### 6.3 输入

- 局部状态/观测特征
- 小规模量子特征映射电路

### 6.4 第一轮量子设置建议

- `4-6` qubits
- 浅层量子特征映射
- 小批样本
- 不接入完整 LETKF 主循环

### 6.5 需要检查的内容

不是先看最终分数，而是先看：

1. 核矩阵是否非负
2. 核矩阵是否不是常数矩阵
3. 核矩阵是否对不同局部状态有反应
4. 核矩阵是否大体平滑
5. 核矩阵是否近似半正定

### 6.6 输出

需要保存：

- 核矩阵热图
- 核矩阵特征值分布
- 不同时间步/不同局部样本的核矩阵对比
- 与经典代理核的对比图

### 6.7 通过标准

只有当量子核满足以下大部分要求时，才算通过：

- 有结构
- 有区分能力
- 不明显退化
- 不明显比经典代理核更乱

### 6.8 止损条件

若量子核：

- 几乎是常数；
- 输入变化时几乎不变；
- 明显不如经典代理核稳定；

则先不要接入 LETKF，而应先重设量子特征映射设计。

---

## 七、实验 4：量子核接入观测加权版 LETKF

### 7.1 目标

在最小改动下验证量子核能否进入 LETKF 局部分析公式。

### 7.2 接入方式

优先接入：

```math
R_i^{-1} \rightarrow {W_i^{(q)}}^{1/2} R_i^{-1} {W_i^{(q)}}^{1/2}
```

而不是一开始就改：

```math
P^f \circ K_q
```

### 7.3 原因

因为观测加权版：

- 更稳；
- 更好解释；
- 更容易定位数值问题；
- 更适合做第一轮 ablation。

### 7.4 对照组

必须至少同时保留：

1. 固定经验局部化版
2. 经典代理核版
3. 量子核版

### 7.5 输出

整理如下表格：

| 方案 | train RMSE | test_1 RMSE | train MAE | test_1 MAE | 是否稳定 | 备注 |
|------|------------|-------------|-----------|------------|----------|------|

并补充：

- 分时间步 RMSE 对比
- 分维度 RMSE 对比
- 若干局部权重可视化

### 7.6 通过标准

若满足以下任一条件，则量子核路线成立：

1. 量子核版稳定优于固定经验局部化；
2. 量子核版不输经典代理核太多，但可解释性明显更强；
3. 量子核版在某些局部流型或时间段有明确优势。

### 7.7 止损条件

若量子核接入后：

- 数值明显不稳定；
- 收益极小；
- 明显不如经典代理核；

则当前阶段不再继续扩大实现复杂度。

## 八、每一步实验结束后要立即写的记录

每个实验完成后，都必须立刻补 4 项内容：

1. **实验目标**
2. **主要结果**
3. **是否支持主线判断**
4. **下一步继续 / 止损决定**

建议统一写进：

- `jindu.md`

这样可以保证后续所有方法选择都是可追溯的，而不是凭印象回忆。

## 九、谁先做什么

如果马上开始执行，建议按下面顺序分工：

### 第一优先

- 跑完 `实验 1`
- 明确固定局部化是否真是瓶颈

### 第二优先

- 做 `实验 2`
- 先验证经典代理核有没有价值

### 第三优先

- 只有实验 2 成立，才做 `实验 3`

### 第四优先

- 只有实验 3 成立，才做 `实验 4`

---

## 十、当前执行建议

如果只允许立刻开始做一件事，那么建议先做：

> **实验 1：固定局部化敏感性实验**

原因：

- 成本最低；
- 与当前主问题最直接相关；
- 能最快回答“局部化到底是不是瓶颈”。

如果实验 1 就不支持主线判断，那么后面很多工作都没必要展开。

---

## 十一、一句话总结

这份执行清单的核心思想不是“赶紧做量子核”，而是：

> **先用最小实验验证每一个关键假设，只有当上一层假设成立时，才允许进入下一层更复杂的实现。**
