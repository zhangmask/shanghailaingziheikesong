## 🎯 测试结果总结

### GPU 加速甜蜜点
- **甜蜜点**: 21 qubits（加速比首次超过 1.0，达到 1.62x）
- **最大加速比**: 4.23x @ 24 qubits
- **32 qubits 时 CPU 内存不足（OOM），GPU 仍可运行**

### CPU vs GPU 性能对比

| qubits | GPU | CPU | 加速比 |
|--------|-----|-----|--------|
| 14 | 0.0025s | 0.0020s | 0.78x |
| 20 | 0.0092s | 0.0072s | 0.79x |
| 24 | 0.0258s | 0.1089s | **4.23x** |
| 28 | 0.3848s | 1.3154s | **3.42x** |
| 32 | 0.0023s | ❌ OOM | - |

### 其他测试结果

| 测试项 | 状态 | 结果 |
|--------|------|------|
| 单进程多卡 16q | ✅ | 平均 0.0101s |
| 4卡并行测试 | ✅ | 平均 0.0113s |
| H2 分子基态能量 | ✅ | -1.1373 Ha（精确） |
| tensor_network GPU | ❌ | 依赖 NV cutensor |
| VQE GPU 算法 | ❌ | V1/V2 API 不兼容 |

### 关键结论

1. **21 qubits 以上才值得用 GPU**，否则 CPU 更快
2. **24 qubits 是最佳加速点**（4.23x），大规模电路 GPU 优势显著
3. **tensor_network 需要沐曦适配 cuTensor**，当前无法使用
4. **qiskit-nature + pyscf 可正常工作**，H2 分子计算精确
5. **Docker 容器需要 --network host** 才能安装额外 Python 包

### 测试 5: tensor_network GPU 报错排查

```python
backend = AerSimulator(method="tensor_network", device="GPU")
# 后端创建成功
qc = QuantumCircuit(10)
# ...
result = backend.run(qc, shots=100).result()
# RuntimeError: Invalid combination of simulation method and device, 
# "tensor_network" only supports "device=GPU"
```

**排查结论**：
- **问题确认**：即使指定 `device="GPU"`，仍然报错说"只支持 GPU"
- **根本原因**：沐曦适配版的 tensor_network 后端依赖 NV 的 cutensor 库
- **无法通过配置解决**：需要沐曦适配 cutensor 或使用替代方案
- **建议**：需要沐曦提供 cuTensor 的 MACA 版本适配

### 测试 6: qiskit-nature + pyscf H2 分子基态能量计算

```
安装 pyscf: 成功（通过内部源 nexus.zs.shaipower.online）
H2 ground state energy: -1.1373060357534008 Ha
精确参考值 (FCI): -1.1373 Ha
误差: < 0.001%
```

**结论**：qiskit-nature + pyscf 在 Docker 容器内可正常工作，H2 分子基态能量计算结果精确

**注意事项**：
- 需要 `--network host` 让容器访问内部 pip 源
- 每次运行需重新安装 pyscf（容器不持久化）
- 兼容性补丁（BaseEstimator → BaseEstimatorV1）已验证有效

### 测试 7: qiskit-algorithms VQE 算法测试

```
VQE with GPU backend for H2
Qubit operator qubits: 4
Error: BaseEstimatorV1.run() missing 1 required positional argument: 'observables'
```

**问题分析**：
- qiskit-algorithms 0.4.0 的 VQE 期望 V1 API: `estimator.run(circuits, observables, parameters)`
- qiskit_aer.primitives.Estimator 的实际实现不兼容
- **这是 qiskit 生态系统的版本碎片化问题**

**解决方案**：
- 方案1：降级 qiskit-algorithms 版本
- 方案2：使用 qiskit_algorithms.optimizers 直接调用 AerSimulator
- 方案3：等待沐曦适配更新
