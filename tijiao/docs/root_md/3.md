# 3号人员测试结论

## 📊 第二轮深度测试总结

### 量子算法测试 (8.1-8.3)

- **QFT**: 14 qubits 时 GPU 略优于 CPU (1.10x)
- **Grover**: 14 qubits 时 GPU 加速最好 (1.74x)
- **GHZ**: 20 qubits 时 GPU 才略微超过 CPU (1.09x)

### 电路特性测试 (9.1-9.2)

- **电路深度是关键因素**: 深度 50 时 GPU 加速达 2.55x
- 不同拓扑结构对加速比影响不大（1.37-1.39x）

### Shots 数量测试 (10.1)

- Shots 数量对 GPU 加速比影响不大
- GPU 主要加速状态向量模拟，采样过程开销相对较小

### 噪声模拟测试 (11.1)

- **重大发现**: 噪声模拟会带来 38-70x 的性能开销
- 这是密度矩阵方法的核心瓶颈

### 后端对比测试 (14.1-14.2)

- **density_matrix 方法极慢**: 比 statevector 慢 6-90 倍（14 qubits 时差距 90 倍）
- GPU/CPU 结果一致性好：最大差异仅 0.16%

## 💡 建议

1. **优先使用 statevector 方法**: 性能最好，GPU 加速效果最明显
2. **避免不必要的噪声模拟**: 除非必须考虑噪声影响
3. **14+ qubits 时考虑 GPU**: 小电路 GPU 启动开销大
4. **深度电路优先使用 GPU**: 深度越深，GPU 优势越明显
