# 实验2：经典代理核实验

> 更新时间：2026-05-16
> 实验目的：在不引入量子模块的前提下，先验证“数据驱动观测权重”是否有潜力替代固定经验局部化，为后续量子核路线提供前置判断。

## 一、实验目标

本实验对应 [实验执行清单.md](file:///d:/Desktop/laingzimuxi/%E5%AE%9E%E9%AA%8C%E6%89%A7%E8%A1%8C%E6%B8%85%E5%8D%95.md) 中的 `实验 2`，要回答的问题是：

> 经典数据驱动代理核，是否能优于固定经验局部化？

如果这一条都不成立，那么后面继续推进量子核替代路线时就必须非常谨慎，因为那会说明：

- 问题不只是“有没有数据驱动”；
- 而是“权重结构本身如何设计”；
- 更进一步说，量子核也不能只是把一个粗糙的经典权重公式量子化。

## 二、实验设置

### 2.1 方法定位

本实验不是重写整个 `LETKF`，而是做**最小替换**：

- 保留 `LETKF` 主骨架；
- 保留预报步骤；
- 保留集合子空间更新结构；
- 只替换观测加权项。

也就是把原来固定局部化里“半径内全等权”的做法，改成：

- `fixed`：固定等权基线
- `corr`：扰动相关性核
- `innov`：创新幅值核
- `hybrid`：相关性 + 创新 + 软距离混合核

### 2.2 固定参数

- `ens_size = 40`
- `infl = 1.02`
- `obs_std = 0.5`
- `dt = 0.05`
- `forcing = 8.0`
- `seed = 2`
- `support_radius = 20`

说明：

- `support_radius = 20` 用作统一支持窗口；
- 在窗口内部，不同代理核用不同权重替代固定等权；
- 这样能保证对比尽量集中在“权重结构”本身，而不是混入更多变量。

### 2.3 数据集

- 训练集：`lorenz96_train.csv`
- 测试集：`lorenz96_test_1.csv`

### 2.4 代理核定义

#### 1. `fixed`

固定等权版本，作为本实验内部对照基线：

```math
w_{ij} = 1
```

它本质上对应：

- 固定支持窗口内全部观测等权参与；
- 与此前 `loc_radius = 20` 的基线一致。

#### 2. `corr`

用局部状态扰动与局部观测扰动的相关性定义权重：

```math
w_{ij} \propto |\mathrm{corr}(x'_i, y'_j)|
```

实验中用绝对相关系数并设置下限裁剪。

#### 3. `innov`

用 innovation 的大小决定权重：

```math
w_{ij} \propto \exp\left(-\frac{|d_j|}{s}\right)
```

其中：

- `d_j` 是局部 innovation；
- `s` 是由局部 innovation 标准差和观测噪声共同决定的尺度。

#### 4. `hybrid`

混合核把三部分合起来：

- 扰动相关性
- innovation 权重
- 软距离衰减

形式上可理解为：

```math
w_{ij} \propto (\alpha + \beta \cdot \mathrm{corr}_{ij}) \cdot \mathrm{innov\_weight}_j \cdot \mathrm{dist\_weight}_{ij}
```

## 四、结果现象

### 4.1 当前代理核没有赢过固定局部化

这是本实验最直接的结论。

三种经典代理核：

- `corr`
- `innov`
- `hybrid`

在训练集和测试集上都明显落后于固定等权基线。

这说明：

> 至少在当前这版设计下，简单把“相关性”或“innovation”直接塞进观测权重，并不能自动带来更好的同化效果。

### 4.2 `corr` 核相对最接近，但仍明显落后

三种代理核里，相对最好的是：

- `corr`

但它仍然明显差于固定局部化：

- `fixed test_1 RMSE = 0.1287279125`
- `corr test_1 RMSE = 0.1436237481`

这说明：

- 扰动相关性本身确实比纯 innovation 更有信息；
- 但“直接按相关性缩放权重”这件事还不够好。

### 4.3 `innov` 与 `hybrid` 表现更差

`innov` 和 `hybrid` 都比 `corr` 更差，尤其在 `test_1` 上退化明显：

- `innov test_1 RMSE = 0.1820804562`
- `hybrid test_1 RMSE = 0.1879526914`

这反映出一个重要问题：

> innovation 大小本身并不适合被直接解释成“观测可信度”。

原因可能包括：

1. innovation 大可能意味着观测带来新信息，不一定应该降权；
2. innovation 里同时混有模型误差、背景误差和观测误差；
3. 直接按 innovation 衰减权重，容易把真正有价值的修正信息压掉。

### 4.4 混合核并没有自然变强

直觉上会觉得：

- 把距离、相关性、innovation 混起来，也许更稳；

但结果并不是这样。

这说明：

> 多个经验因子相乘，不会自动形成更合理的核，反而可能把错误偏好叠加起来。

也就是说，后面如果要设计量子核，也不能只是：

- 把更多因素乘起来；
- 或者把经典启发式塞进量子电路；

否则大概率只是更复杂，但不更好。

## 五、对主线假设的判断

本实验对应的核心假设是：

> 假设 2：数据驱动相似度权重可以优于固定经验局部化。

当前判断：**暂不支持。**

但这里要注意，“暂不支持”不是说整条路线彻底失败，而是更具体地说：

> **当前这版经典代理核设计，不支持继续直接往下量子化。**

也就是说，本实验否定的不是“大方向”，而是：

- 过于直接的相关性权重；
- 过于粗糙的 innovation 权重；
- 简单相乘式的混合核。

---

## 六、这次实验真正说明了什么

这次实验其实很有价值，因为它排除了一个常见误区：

> 只要把经典局部化换成某种“数据驱动权重”，效果就会自动更好。

结果表明，并不是这样。

这对后续工作有三个直接启发。

### 6.1 后续核设计不能只看单步局部统计

当前代理核只用了：

- 局部扰动相关性
- 单步 innovation
- 软距离

这些特征都太“即时”了。

但 Lorenz96 是混沌动力系统，真正有效的局部权重可能需要看：

- 时间上的一致性
- 局部窗口的动态演化
- 多步统计结构

### 6.2 权重结构不能只做乘法拼接

当前 `hybrid` 的失败说明：

- 简单地把多个启发式因子相乘，并不会自然得到一个好核；
- 反而容易过度抑制某些本应保留的观测。

因此后续更合理的方向，不是继续堆更多手工因子，而是：

- 重新定义特征表示；
- 重新定义相似度空间；
- 让核结构从表示层开始改变。

### 6.3 这反而给量子核路线提了更高要求

如果经典代理核已经轻松赢过固定局部化，那么量子核只需要“更强一些”即可。

但当前情况不是这样。

现在更严肃的要求是：

> 量子核不能只是“经典代理核的量子版”，而必须在特征表示或相似度结构上真正提供不同的表达能力。

这恰恰更支持我们前面已经写下的判断：

- 量子价值应该落在**特征映射**上；
- 而不是落在“把经验权重量子化”上。

---

## 七、对后续实验的影响

这次实验之后，后续路线需要稍微收紧。

### 7.1 不建议直接做“经典代理核的量子翻版”

因为经典这一版已经说明：

- 直接用 `corr / innov / hybrid` 权重，不足以替代固定局部化。

所以后面如果做量子核，不能简单对应成：

- 量子相关性核
- 量子 innovation 核
- 量子混合乘法核

那样大概率不会有本质改进。

### 7.2 下一步更合理的是“量子核结构实验”

当前更值得推进的是：

- 先做小规模量子特征映射；
- 先观察量子核矩阵本身有没有结构；
- 看它是否提供了与经典代理核不同的相似度几何。

也就是说，后续 `实验 3` 的重点不应该是：

- 直接拼结果；

而应该是：

- 先看核矩阵是否非平凡；
- 是否有区分能力；
- 是否可能对应更丰富的局部表示。

---

## 八、当前结论

本次实验最重要的一句话是：

> 固定经验局部化虽然有明显局限，但这并不意味着任何“数据驱动权重”都会更好；当前这版经典代理核并未优于固定局部化，说明后续若继续推进量子核路线，必须把重点放在更强的特征表示与相似度结构上，而不能只是把粗糙的经典权重公式量子化。

这意味着：

- `实验 1` 支持了“局部化值得研究”；
- `实验 2` 则进一步说明：
  - “怎么研究局部化”比“要不要研究局部化”更关键。

## 十、结构补充分析

### 10.1 为什么要补这一部分

`实验执行清单.md` 里对 `实验 2` 的要求不只是误差表，还包括：

- 若干时间步的权重矩阵热图；
- 若干局部点的观测权重曲线。

这部分在第一次跑完主结果时还没落盘，所以本次续做专门补上，目的不是改结论，而是让“为什么这些代理核会退化”有更直接的结构证据。

### 10.2 补充分析设置

本次结构分析采用与 `实验 2` 一致的参数设置：

- `ens_size = 40`
- `infl = 1.02`
- `obs_std = 0.5`
- `support_radius = 20`
- `seed = 2`

并在训练集上选取有观测的三个代表时间步：

- `t = 0`
- `t = 500`
- `t = 998`

重点查看：

- 中间时刻 `t = 500` 的完整权重矩阵热图；
- 固定状态维度 `i = 10` 在 `t = 500` 的观测权重曲线；
- 早期 / 中期 / 后期三个时刻的权重统计量。

### 10.3 结构统计结果

代表性统计如下：

| kernel | time_step | min | max | mean | std |
|--------|-----------|-----|-----|------|-----|
| fixed | 500 | 1.0000 | 1.0000 | 1.0000 | 0.0000 |
| corr | 500 | 0.2568 | 4.1792 | 1.0000 | 0.7489 |
| innov | 500 | 0.2214 | 1.9253 | 1.0000 | 0.5122 |
| hybrid | 500 | 0.2877 | 7.8685 | 1.0000 | 1.1008 |

可见：

- `fixed` 完全等权，没有结构波动；
- `corr` 已经出现明显结构，但波动幅度仍相对可控；
- `innov` 会按残差大小压低一部分观测；
- `hybrid` 的波动最强，说明多个经验因子叠加后会形成更尖锐的权重分配。

### 10.4 新增结构证据支持了什么判断

这次补充最关键的是把“误差退化”与“权重形状”对应了起来。

1. `fixed` 虽然朴素，但结构最稳  
   在 `support_radius = 20` 时，它本质上就是全局等权。对于当前 40 维 Lorenz96 小规模任务，这种简单结构反而避免了过度抑制。

2. `corr` 不是没信息，而是“有信息但不够稳”  
   从热图和曲线看，`corr` 已经能够分辨不同观测的重要性，所以它比 `innov / hybrid` 更接近固定基线；但这种单步相关性仍不足以稳定替代等权。

3. `innov` 的问题是过于直接地把残差解释成可信度  
   结构图上能看到它会对部分观测持续降权，而 innovation 大并不一定意味着“应该少用”，这会把一些真正有修正价值的信息压掉。

4. `hybrid` 的问题是经验偏好叠加过强  
   它的标准差最大，权重峰谷也最尖锐，说明“相关性 + innovation + 距离”并没有自然形成更合理的核，反而更容易把错误偏好叠加起来。

### 10.5 对后续实验的意义

补完结构分析后，`实验 2` 的结论更完整了：

- 不只是“经典代理核没赢”；
- 而是更具体地说明了“它们为什么没赢”。

因此后续如果继续做量子核，重点应当放在：

- 更强的特征表示；
- 更合理的相似度几何；
- 而不是把现有 `corr / innov / hybrid` 公式直接量子化。
